Phase 1 — X01 in-game (the big one)

This is 80% of session time. Build it as one widget tree top-to-bottom matching the mock:

1

Top bar (BroadcastBar)

Row with BACK button (Navigator.pop), LIVE pill, meta text ("501 · D-OUT · R7 · LEG 1"), settings icon. Use Border(bottom: BorderSide(color: dividerColor)).

2

Active player block (NowThrowing)

Listens to current-player state. Amber gradient header, big avatar, display-font name, score on the right. Three dart pips (filled = thrown). Bottom row: turn delta, last dart, average, checkout suggestion. Most reactive part of the screen — wire it to your existing turn state.

3

Other-players strip (OtherPlayersStrip)

Horizontal flex of 3 small cells (Row with Expanded children). Name + score + last-turn delta. This is the new piece — it's just a denormalized read of game state.

4

Dartboard (DartboardInput)

Reuse your existing input widget. Just make sure it gets most of the vertical space — wrap it in Expanded with center alignment.

5

Footer (TurnActionBar) — UNDO + MISS + END TURN

UNDO is the new addition you flagged. Implement as: keep a stack of TurnDelta records (player id, score before, darts thrown). UNDO pops the top and restores. Disable when stack is empty. MISS records a 0. END TURN commits and advances. Order left-to-right: UNDO, MISS, spacer, END TURN (primary, orange).

Phase 2 — Player setup

  1. Replace the scrolling list with a GridView.count(crossAxisCount: 3) of PlayerCard tiles.
  2. Selection shows an amber circle with the join-order number (1, 2, 3…) in the top-right corner. This is the cheapest way to show order without a separate "drag to reorder" UI.
  3. Top toolbar: mode chip (X01 selected), start-score chips (301 / 501 / 701), out-rule chip.
  4. Below toolbar: "N SELECTED" pill on the left, "🎲 RANDOMIZE" button on the right. Randomize shuffles the order numbers, not the grid positions — so the user keeps spatial memory of where each friend's tile is.
  5. Footer: + NEW, CLEAR, spacer, START → (primary).

Phase 3 — Post-game

  1. Winner block — same gradient header pattern as NowThrowing. Big avatar, big name, rating delta pill (read from your rating service), darts count.
  2. Comparison matrix — Table widget. Cols: Player, Best, Avg, Darts, Δ. Bold the winning cell in each column with amber. Row order = finishing order.
  3. Score progression chart — small line chart, one polyline per player. Use fl_chart with a minimal config (no axes, no labels — just dashed gridlines). Snapshot per round from your existing turn log.
  4. Highlights — list of 2-4 strings generated from the turn log: ton+ throws, busts, the winning checkout. Computable in <50 lines of Dart.
  5. Footer — HOME (ghost), UNDO LAST (outline — for "wait, that wasn't a checkout"), spacer, REMATCH → (primary).
